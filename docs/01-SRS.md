# 软件需求规格说明书（SRS）

## 1. 项目概述

### 1.1 项目名称
代码片段速查本（Code Snippet Vault）

### 1.2 项目目标
为开发者提供一个简洁、优雅、高效的代码片段管理工具。通过极致的Apple风格UI体验，帮助用户快速管理、搜索、复制常用代码片段，显著提升开发效率。

### 1.3 目标用户

| 用户类型 | 特征 | 核心诉求 |
|---------|------|---------|
| 前端/后端开发者 | 日常需要复用大量代码模板 | 快速检索、一键复制、标签管理 |
| 计算机专业学生 | 学习过程中积累代码示例 | 分类整理、语言筛选、简洁易用 |
| 技术团队成员 | 团队内共享常用代码块 | 统一存储、快速查找、标准化管理 |

### 1.4 项目范围

**包含（In Scope）：**
- 代码片段的增删改查（CRUD）
- 多维度搜索（关键词、语言、标签）
- 标签筛选与管理
- 一键复制代码到剪贴板
- Apple风格极致UI体验
- Docker一键部署

**不包含（Out of Scope）- MVP阶段：**
- 用户登录与权限管理
- 代码片段分享功能
- AI辅助代码生成
- 云端数据同步
- 多用户协作
- 代码执行与预览

## 2. 功能需求

### 2.1 用户故事

#### US-001 浏览代码片段
> 作为一名开发者，我希望在首页看到所有代码片段的卡片列表，以便快速浏览我的代码库。

**验收标准：**
- 首页以卡片网格形式展示所有片段
- 每张卡片显示标题、编程语言、标签列表和创建时间
- 卡片hover时有微妙的 elevation 提升效果
- 支持响应式布局（桌面3列、平板2列、手机1列）
- 空状态时显示精美的引导插画

#### US-002 搜索代码片段
> 作为一名开发者，我希望通过关键词搜索代码片段，以便快速找到我需要的代码。

**验收标准：**
- 顶部有全局搜索栏，支持实时搜索
- 搜索范围覆盖标题、描述、代码内容
- 搜索时显示 loading 状态（Apple风格进度指示器）
- 搜索结果高亮匹配关键词
- 支持搜索防抖（300ms）
- 无结果时显示友好提示

#### US-003 按标签筛选
> 作为一名开发者，我希望按标签筛选代码片段，以便查看同一类别的代码。

**验收标准：**
- 标签以 pill 形式展示在搜索栏下方
- 点击标签切换选中/未选中状态
- 支持多标签组合筛选（AND逻辑）
- 选中标签有视觉反馈（背景色+阴影）
- 显示当前筛选结果数量

#### US-004 按编程语言筛选
> 作为一名开发者，我希望按编程语言筛选，以便查看特定语言的代码片段。

**验收标准：**
- 提供语言下拉选择器
- 支持常见语言：JavaScript、TypeScript、Python、Java、Go、Rust、CSS、HTML、Shell、SQL等
- 语言选择器采用Apple风格下拉菜单
- 可清空语言筛选条件

#### US-005 创建代码片段
> 作为一名开发者，我希望创建新的代码片段，以便保存我常用的代码。

**验收标准：**
- 点击"新建"按钮打开SlideOver面板
- 表单字段：标题（必填）、编程语言（必填）、标签（可选，支持多选）、描述（可选）、代码内容（必填）
- 代码编辑器支持语法高亮
- 表单验证实时反馈（Apple风格错误提示）
- 创建成功后显示 Toast 提示并自动关闭表单
- 新片段出现在列表首位

#### US-006 查看代码片段详情
> 作为一名开发者，我希望查看代码片段的完整内容和详情，以便确认这是我需要的代码。

**验收标准：**
- 点击卡片打开详情面板/页面
- 详情页展示完整字段信息
- 代码区域支持语法高亮显示
- 显示创建时间和最后更新时间
- 详情页有关闭/返回按钮

#### US-007 编辑代码片段
> 作为一名开发者，我希望编辑已有的代码片段，以便更新代码或修正信息。

**验收标准：**
- 在详情页提供编辑入口
- 编辑表单与创建表单保持一致
- 预填充现有数据
- 保存后即时更新列表和详情
- 编辑成功后显示 Toast 提示

#### US-008 删除代码片段
> 作为一名开发者，我希望删除不再需要的代码片段，以便保持代码库整洁。

**验收标准：**
- 在详情页提供删除入口
- 删除前弹出确认对话框（Apple风格Alert）
- 确认后永久删除
- 删除成功后返回列表页并显示 Toast
- 列表自动移除已删除项

#### US-009 一键复制代码
> 作为一名开发者，我希望一键复制代码到剪贴板，以便直接粘贴使用。

**验收标准：**
- 代码区域提供复制按钮
- 点击后代码复制到系统剪贴板
- 复制成功后有视觉反馈（按钮图标变化+Toast）
- 支持复制完整代码内容

## 3. 非功能需求

### 3.1 性能需求

| 指标 | 要求 | 说明 |
|------|------|------|
| 首屏加载时间 | < 1.5s | 包含所有静态资源 |
| 搜索响应时间 | < 300ms | 本地运行时 |
| 列表渲染 | < 100ms | 50条数据以内 |
| 交互反馈 | < 50ms | 按钮点击、hover等 |
| 动画帧率 | 60fps | 所有过渡动画 |

### 3.2 兼容性需求

| 平台 | 支持版本 |
|------|---------|
| Chrome | 最新2个主版本 |
| Safari | 最新2个主版本 |
| Firefox | 最新2个主版本 |
| Edge | 最新2个主版本 |

### 3.3 UI/UX需求

**整体风格：Apple官网级设计品质**

- **视觉风格**：极简主义、大量留白、精致的圆角（12-16px）、微妙的阴影层次
- **色彩系统**：以白色/浅灰为底，深色文字，蓝色作为品牌强调色，无多余装饰色
- **字体排版**：使用系统字体栈（-apple-system, BlinkMacSystemFont, "Segoe UI"），清晰的字体层级
- **动效设计**：所有状态变化都有平滑过渡（0.2-0.4s, ease-out），滚动触发动画，按钮有按压反馈
- **组件风格**：自研UI组件，不依赖任何第三方UI库，确保视觉一致性
- **交互细节**：复制按钮hover放大、卡片hover提升、输入框focus发光效果

### 3.4 可维护性需求

- 前后端分离架构
- RESTful API设计
- 完整的TypeScript类型覆盖
- ESLint + Prettier代码规范
- Docker Compose一键启动

### 3.5 安全需求

- API输入验证与消毒
- 防止XSS攻击（代码内容转义显示）
- CORS配置
- 服务端错误信息不暴露敏感信息

## 4. 界面原型描述

### 4.1 页面结构

```
┌─────────────────────────────────────────┐
│  Header（Logo + 全局搜索栏 + 新建按钮）   │
├─────────────────────────────────────────┤
│  Filter Bar（标签筛选 + 语言筛选）        │
├─────────────────────────────────────────┤
│                                         │
│         代码片段卡片网格区域              │
│                                         │
│  ┌─────────┐  ┌─────────┐  ┌─────────┐ │
│  │ 卡片 1  │  │ 卡片 2  │  │ 卡片 3  │ │
│  └─────────┘  └─────────┘  └─────────┘ │
│                                         │
├─────────────────────────────────────────┤
│  Footer（简洁版权信息）                  │
└─────────────────────────────────────────┘
```

### 4.2 详情/编辑面板

采用侧边滑出面板（Slide-over）或模态对话框形式，背景有毛玻璃遮罩效果。

```
┌────────────────────┬──────────────────────┐
│                    │  标题输入框          │
│   列表页背景        │  语言选择器          │
│   （毛玻璃遮罩）    │  标签输入区          │
│                    │  描述文本域          │
│                    │  代码编辑器          │
│                    │  [保存] [取消]       │
└────────────────────┴──────────────────────┘
```
