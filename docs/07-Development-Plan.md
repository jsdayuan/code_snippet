## 3. 风险与应对

| 风险 | 可能性 | 影响 | 应对措施 |
|------|--------|------|----------|
| 自研组件开发量超预期 | 中 | 工期延迟1天 | 优先开发核心组件（Button/Input/Card/SlideOver），次要组件可简化 |
| PrismJS语法高亮配置复杂 | 低 | 阻塞30min | 提前准备常用语言的theme配置 |
| MongoDB全文搜索性能不佳 | 低 | 搜索体验差 | 预留方案：改为正则搜索，或引入Elasticsearch（后续迭代） |
| CSS动效性能问题 | 低 | 卡顿 | 使用transform和opacity（GPU加速），避免触发layout |

## 4. 每日检查点

### 第1天结束检查
- [ ] `docker compose up` 能启动三个容器
- [ ] 前端页面能正常打开（空白页即可）
- [ ] 后端 `/api/snippets` 返回空数组或种子数据

### 第2天结束检查
- [ ] 后端6个API全部通过测试
- [ ] 15个基础组件能在测试页展示

### 第3天结束检查
- [ ] 完整功能可用（CRUD + 搜索 + 筛选 + 复制）
- [ ] 无明显bug

### 第4天结束检查
- [ ] Docker Compose 一键部署成功
- [ ] README 完整，新用户5分钟内能跑起来
